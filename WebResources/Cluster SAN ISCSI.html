Introduction

Dans cet article, nous allons parler de La technologie ISCSI (Internet Small Computer System Interface) qui est un protocole réseau de stockage basé sur le modèle TCP/IP. Le principe de base consiste à encapsuler des commandes SCSI dans des paquets IP transmis entre un hôte et une unité de disque. Comme les paquets IP peuvent être perdus (et/ou) retransmis, ils peuvent très bien ne pas arriver dans l'ordre d'émission. Le protocole iSCSI doit donc conserver une trace de la séquence de transmission de commandes SCSI. Les commandes sont placées dans une file d'attente dans l'ordre d'émission.

La technologie iSCSI a initialement été développée par IBM et a ensuite été soumise à l'IETF (Internet Engineering Task Force). Le standard a été publié par le comité IP Storage Working Group en août 2002.

On peut identifier deux fonctions principales dans la technologie iSCSI. La première est la fonction target. C'est un système simple qui possède le volume de stockage à publier sur le réseau IP. Ce système peut être matériel ou logiciel. Dans le cas de ces travaux pratiques, il s'agit d'un poste de travail utilisant son second disque dur ou bien un fichier comme unité de stockage SAN. La seconde fonction est baptisée initiator. Elle correspond au «client» qui utilise le volume de stockage réseau. Dans le contexte de ce document, l'autre poste de travaux pratiques joue ce rôle de client.

Fondamentalement, iSCSI est un protocole de la famille Storage Area Network (SAN). Le client ou initiator accède à une unité de stockage en mode bloc. Ce mode de fonctionnement est quasi identique à la technologie Fibre Channel. Le type de réseau constitue la principale différence entre ces deux technologies. La technologie iSCSI s'appuie sur TCP/IP alors que Fibre Channel comprend une définition de réseau propre (FC) qui nécessite des équipements spécifiques.

Pourquoi utiliser iSCSI ?

Ces dernières années, la technologie iSCSI gagne en popularité relativement à son ainée pour plusieurs raisons. Notamment, le prix des configurations iSCSI peut être bien meilleur marché qu'avec la technologie Fibre Channel. Si les performances d'un réseau au Gigabit Ethernet suffisent, un réseau de stockage iSCSI devient très attractif.

Mise en place d'un Cluster ISCSI avec Réplication

Prérequis

Il faut créer deux machines Windows Server 2012 R2. (RAM : 4096MB, disque 1 : 20 GO, disque 2 : 500 GO) choisir Windows Server 2012 R2 avec interface graphique et créer deux partitions.

Esuite, il faut changer le nom de la machine (clique droit sur CE PC->Propriétés->Modifies les paramètres->Modifier).

Enfin, il faut attribuer une IP Fixe (Panneau de configuration\Réseau et Internet\Connexions réseau).

Installation des rôles et création de l’ISCSI Target

Ces opérations devront être faites de la même manière pour le serveur SRVSAN02.

Afin de créer le target iSCSI, se diriger vers la MMC (Microsoft Management Console) et ajouter un nouveau rôle au serveur SRVSAN01.

Aller dans l’Assistant Ajout de rôle et de fonctionnalités et cocher :

- Serveur de fichier : Permet aux utilisateurs d’accéder aux fichiers stockés dedans.

- BranchCache pour fichiers réseau : Optimise la Bande Passante sur le réseau.

- Déduplication des données : Economie de l’espace disque en ne stockant qu’une seule copie des données identiques sur le volume.

- Dossier de travail : Permet une utilisation des fichiers de travail depuis de nombreux appareils, et plusieurs utilisateurs simultanés depuis le réseau ou depuis internet.

- Fournisseur de stockage cible ISCSI : Permet aux applications des serveurs d’accéder aux volumes.

- Réplication DFS : Permet une réplication des données sur d’autres serveurs maitre pour garder la disponibilité des données.

- Serveur Cible ISCSI : Fournit des services et des outils de gestion.

Ensuite, il faut lancer l’installation des rôles.

Une fois l’installation terminée, dans la console Gestionnaire de serveur, accéder au service de stockage et de fichiers pour paramétrer le ISCSI TARGET.

Choisir l’espace de stockage, qui sera utilisé comme Volume de stockage ISCSI Storage.

Indiquer le nom du disque virtuel ISCSI.

Il faut choisir Taille fixe : La taille du disque virtuel est fixée et ne pourra être changée ultérieurement. Ce type de disque fournit de meilleures performances sur les applications les plus gourmandes en activité de disque.

Affecter un nom au disque dur virtuel ISCSI. Choisir quels serveurs seront autorisés à accéder à l’ISCSI STORAGE.

Les serveurs choisis seront les serveurs qui auront droit d'accéder aux ISCSI TARGETS depuis un ISCSI INITIATOR

En suite il faut sécuriser l’accès á l’ISCSI STORAGE avec la sécurité CHAP.

Ensuite créer le ISCSI STORAGE.

La cible ISCSI STORAGE est créée.

Création du cluster SAN avec basculement

Il faut Aller dans Gestionnaire du serveur puis ajouter des rôles et des fonctionnalités, ensuite Installer la fonctionnalité clustering avec basculement, ce qui va permettre aux données d’être accessibles sans interruption en cas de panne d’un serveur.

Une fois l’installation terminée, se rendre dans Gestionnaire du cluster de basculement Puis créer le cluster.

Ensuite lancer le test de validation des serveurs du cluster.

Il faut cliquer sur créer, pour lancer la création du cluster.

Notre Cluster qu'on vient de créer est disponible.

Il ne faut pas oublier de créer le QUORUM pour le Vote.

Configuration de la Réplication ISCSI

Pour que les donnes soient accessibles en cas de problème sur un serveur, il faut mettre en place une réplication des données qui se trouve dans le ISCSI STORAGE entre le SRVSAN01 et SRVSAN02 qui sont en cluster.

Pour cela, il faut aller dans Gestion du système de fichier distribués DFS.

Réplication->Nouveau groupe de réplication :

Choisir groupe de réplication multi-usage

Renseigner le nom du groupe de réplication, le domaine est déjà renseigné

Renseigner les serveurs qui formeront le groupe de réplication

Il faut Choisir la topologie de réplication Maile Pleine ce qui permettra une réplication entre les deux serveurs qui contiennent les ISCSI Storage dans les deux sens en cas de panne du serveur maitre.

Il faut Mettre la planification de la réplication en 24h/24 et 7j/7.

Puis Mettre la bande passante á complète pour que la réplication soit le plus rapide possible.

Il faut Choisir le serveur principal du cluster, pour définir le sens de la première réplication.

Puis Définir le ISCSI Storage pour la réplication.

Activer la synchronisation avec les membres du cluster.

Puis créer.

A la fin, nous pouvons voir toutes les options que nous avions paramétré avant.

La création de la réplication se termine avec succès.

Après la création de notre réplication nous pouvons accéder aux informations ci-dessous.

Connexion de l'ISCSI INITIATOR avec l'ISCSI TRAGET

Sur le poste client, aller Dans Gestionnaire du serveur, aller dans outils puis Initiateur ISCSI. Entrer l’adresse IP du SRVSAN01 connecter le iscsi target (iscsi Storage), ensuite faire la même manipulation pour connecter le iscsi Storage du SRVSAN02.

Nous pouvons constater que le ISCSI Storage01 est détecté et connecté.

Ensuite les ISCSI Storage apparaitront sur le serveur.

Nous pouvons aussi les voir connectés dans Ordinateur.

Conclusion

La mise en place d’un SAN devra passer par une étude préliminaire afin de trouver la solution qui nous corresponde le mieux.

De plus, le choix d'un SAN ISCSI devra lui aussi faire l’objet d’une étude.

En effet,le ISCSI dans bien des cas suffit et demande un investissement moins lourd contrairement au FIBER CHANNEL, comme nous avions pu le constater lors de la mise en place de notre SAN ISCSI.
