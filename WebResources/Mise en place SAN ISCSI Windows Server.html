Mise en place d'un SAN ISCSI sur Windows Serveur 2012 R2

Le but de cet article est d'expliquer la mise en place d'un SAN ISCSI sur Windows Serveur 2012 R2. Nous verrons en détail chaque étape de la création et configuration d'un SAN ISCSI.

Présentation

Internet Small Computer System Interface (ISCSI) est un protocole de stockage en réseau basé sur le protocole TCP/IP. Il est destiné à relier des installations de stockage de données. C'est un protocole de SAN (Storage Area Network) qui permet de rassembler les ressources de stockage dans un centre de données tout en donnant l'illusion que le stockage est local.

La technologie iSCSI transporte des blocs de données entre un initiateur iSCSI (Initiator), installé sur un serveur, et une cible iSCSI (Target), installée sur un dispositif de stockage.

Pré-requis

Nous aurons besoin d'au moins deux machines avec Windows Serveur 2012 R2 installé.

Installation des rôles nécessaire

Avant toute chose, il faut ajouter depuis notre "Server Manager" (Version en anglais) les rôles suivants :

- File Server

- Data Deduplication

- iSCSI Target Server

- iSCSI Target Storage Provider

Après la sélection des rôles, continuer jusqu'à la page finale de l'installation.

Une fois l'installation des rôles terminée nous pouvons passer à la création des disques virtuels ISCSI.

Création des disques virtuels ISCSI et configuration du Target

Depuis le "Server Manager" accéder à "File and Storage Services" puis à "iSCSI" pour procéder à la création des disques.

Lorsque l'assistant de création de disques virtuels ISCSI est lancé, il faut en premier lieu choisir l'emplacement du disque virtuel.

Puis le nom du disque.

Ensuite choisir la taille de notre disque virtuel.

Après cela il faudra assigner ce disque virtuel à un "ISCSI Target". Pour ce faire, nous allons créer un nouvel ISCSI Target.

En définissant dans un premier temps le nom du Target.

Puis les "ISCSI initiators" qui auront accès à ce target.

Notons qu'il y'a plusieurs méthodes permettant d'identifier les initiators.

La méthode choisie ici étant la deuxième, affiche directement les initiators disponible car une manipulation préalable à été effectuée auprès de ces derniers.

Petite parenthèse pour expliquer cette fameuse manipulation...

Sur chaque initiator que nous souhaitons lier à ce target :

Dans notre "Server Manager", cliquer sur "Tools" puis "ISCSI Initiator".

Un message d'alerte nous permettant de démarrer maintenant et automatiquement après chaque redémarrage le service "Microsoft ISCSI" s'affiche alors.

Nous avons maintenant accès à une fenêtre permettant de gérer les propriétés de notre initiator.

Il faudra à ce niveau entrer l'adresse IP du serveur où se trouve notre target (En cours de configuration) puis cliquer sur "Quick Connect.

Un message apparait donc, nous indiquant qu'il n'y a aucun Target disponible. Ce qui est tout à fait normal car la configuration du target n'est pour le moment pas complète.

Cliquer sur "Done".

On retourne alors sur la fenêtre de départ.

Dans le menu d'en haut, cliquer sur "Discovery". Nous pouvons constater que l'adresse IP saisie précédemment s'est ajoutée ici, le système cherchera donc un target à cette adresse IP.

Pour revenir à la configuration de notre Target, après avoir sélectionné les initiators auquels nous voulons donner accès à ce target il est possible si souhaité d'activer le protocole CHAP ou le Reverse CHAP.

Puis confirmer la création.

Nous observons à la fin que la création du disque virtuel, la création du target, l'attribution d'accès du target aux initiators et l'assignation du disque virtuel au target se sont réalisés avec succès.

Il est possible si souhaité de créer et d'assigner plusieurs disques virtuel par la suite à ce target en suivant le même procédé du début.

Deux autres disques virtuels ont étés ajoutés dans ce cas comme le montre la liste des disques virtuels.

Les disques virtuels et le target sont en statut "Not Connected", il faudra donc une configuration au niveau des initiators...

Configurations au niveau des initiators

Au niveau de la fenêtre de gestion des propriétés de nos initiators, un "refresh" nous permet de remarquer l'apparition de notre Target qui a un statut inactif.

Nous allons donc cliquer sur "Connect" puis sur "ok" pour que la connexion au target soit effective.

La prochaine étape consiste à aller dans le gestionnaire de disque windows pour mettre en ligne les disques virtuels, les initialiser et créer de nouveaux volumes simples.

Ainsi, en retournant sur notre target, après avoir rafraîchis le "Server Manager", nous pouvons constater le changement de statut de nos disques et du target qui est passé de "not connected" à "connected". Ce qui indique que tout est alors bien fonctionnel.
